# UX Analysis Checklist

This checklist provides a systematic framework for evaluating user experiences from a psychology-informed perspective.

## How to Use This Checklist

1. **Review each category** relevant to the page/feature being analyzed
2. **Rate each item** (✅ Good, ⚠️ Needs improvement, ❌ Missing/Poor, N/A Not applicable)
3. **Note specific issues** with line-item details
4. **Prioritize improvements** based on impact and effort
5. **Reference principles.md** for detailed implementation guidance

---

## 1. First Impressions & Aesthetic Appeal

### Visual Design Quality
- [ ] **Aesthetic-Usability Effect**: Does the design look polished and professional?
- [ ] **Visual consistency**: Are colors, typography, and spacing consistent?
- [ ] **White space**: Is there adequate breathing room between elements?
- [ ] **Imagery**: Are images high-quality and purpose-driven?
- [ ] **Brand alignment**: Does the design match brand expectations?

### Immediate Clarity
- [ ] **Value proposition**: Is it clear what this page/product offers?
- [ ] **Visual hierarchy**: Does the design guide eyes to important elements?
- [ ] **First-screen content**: Are key messages visible without scrolling?
- [ ] **Loading experience**: Does the page load quickly and progressively?

**Impact**: High - First impressions strongly influence trust and engagement
**Priority**: Address major aesthetic issues before detailed optimizations

---

## 2. Information Architecture & Navigation

### Cognitive Load Management
- [ ] **Miller's Law**: Are menu/category counts within 5-9 items per level?
- [ ] **Progressive disclosure**: Is complex information revealed gradually?
- [ ] **Chunking**: Is content grouped into digestible sections?
- [ ] **Scanning**: Can users quickly scan and find information?
- [ ] **Search functionality**: Is search available for content-heavy pages?

### Navigation Patterns
- [ ] **Familiarity principle**: Do navigation patterns match user expectations?
- [ ] **Confirmation bias**: Are there multiple paths to the same content?
- [ ] **Breadcrumbs**: Can users understand their location in the hierarchy?
- [ ] **Mobile navigation**: Is navigation adapted appropriately for small screens?

### Content Organization
- [ ] **Visual hierarchy**: Are headings, subheadings, and body text clearly differentiated?
- [ ] **Readability**: Is text size, line height, and contrast appropriate?
- [ ] **Content structure**: Do headings and structure support scanning?

**Impact**: High - Poor IA causes frustration and abandonment
**Priority**: Fix navigation confusion before optimizing conversion elements

---

## 3. Cognitive Load & Usability

### Task Complexity
- [ ] **Intrinsic load**: Is the task as simple as it can be?
- [ ] **Extraneous load**: Is there unnecessary complexity in the interface?
- [ ] **Form design**: Are forms minimal, with only essential fields?
- [ ] **Multi-step flows**: Are complex processes broken into manageable steps?
- [ ] **Goal gradient**: Are progress indicators shown for multi-step tasks?

### Clarity & Understanding
- [ ] **Labels and copy**: Are labels clear without jargon?
- [ ] **Instructions**: Are complex tasks explained simply?
- [ ] **Error messages**: Are errors helpful and actionable?
- [ ] **Feedback**: Do user actions receive immediate, clear feedback?
- [ ] **Help resources**: Is contextual help available when needed?

### Decision Support
- [ ] **Choice paradox**: Are options limited to 3-5 when possible?
- [ ] **Default effect**: Are sensible defaults provided?
- [ ] **Recommendations**: Are suggested or popular choices highlighted?
- [ ] **Comparison tools**: Can users easily compare options?

**Impact**: High - Cognitive overload drives abandonment
**Priority**: Reduce friction in critical user journeys first

---

## 4. Persuasion & Motivation

### Social Proof Implementation
- [ ] **User counts**: Are customer/user numbers displayed where relevant?
- [ ] **Testimonials**: Are real customer testimonials featured?
- [ ] **Reviews/ratings**: Are user reviews prominently displayed?
- [ ] **Case studies**: Are success stories from similar users available?
- [ ] **Trust badges**: Are relevant certifications/awards shown?
- [ ] **Real-time activity**: Is live social proof used appropriately?

### Scarcity & Urgency
- [ ] **Scarcity effect**: Is limited availability shown (if true)?
- [ ] **Urgency**: Are time-sensitive offers clearly communicated?
- [ ] **Authenticity**: Are scarcity/urgency claims verifiable and honest?
- [ ] **Balance**: Is urgency used without creating excessive anxiety?

### Loss Aversion & Framing
- [ ] **Copy framing**: Is copy framed to emphasize benefits or prevent losses?
- [ ] **Risk reduction**: Are guarantees, free trials, or returns highlighted?
- [ ] **Commitment size**: Does the ask start small (foot in the door)?

**Impact**: Medium-High - Effective persuasion increases conversion
**Priority**: Implement after core usability is solid

---

## 5. Engagement & Retention Mechanics

### Motivation Design
- [ ] **Goal gradient effect**: Are progress indicators used in flows?
- [ ] **Gamification**: Are game elements (if used) meaningful and aligned with goals?
- [ ] **Variable reward**: Is content/experience varied to maintain interest?
- [ ] **Endowment effect**: Is ownership language used ("your dashboard")?
- [ ] **Achievements**: Are milestones celebrated appropriately?

### Onboarding & Adoption
- [ ] **Progressive onboarding**: Are users guided through initial setup?
- [ ] **Quick wins**: Can new users experience value quickly?
- [ ] **Tooltip support**: Is contextual help available for unfamiliar features?
- [ ] **Empty states**: Do empty states guide users toward first actions?

### Continued Engagement
- [ ] **Personalization**: Is content tailored to user preferences/behavior?
- [ ] **Notifications**: Are notifications valuable and not overwhelming?
- [ ] **Return triggers**: Are there clear reasons to return?

**Impact**: Medium - Engagement mechanics improve retention
**Priority**: Focus on onboarding first, then ongoing engagement

---

## 6. Trust & Credibility Signals

### Authority & Expertise
- [ ] **Authority bias**: Are credentials, certifications, or awards displayed?
- [ ] **Expert content**: Is content authored by qualified experts?
- [ ] **Professional design**: Does the aesthetic signal competence?
- [ ] **About/team pages**: Is there transparency about who's behind the product?

### Security & Privacy
- [ ] **Security badges**: Are SSL, payment security indicators visible?
- [ ] **Privacy policy**: Is privacy policy easily accessible?
- [ ] **Data handling**: Is data usage clearly explained?
- [ ] **Compliance**: Are relevant compliance indicators shown (GDPR, etc.)?

### Transparency
- [ ] **Pricing clarity**: Are prices and terms clearly stated?
- [ ] **Honest marketing**: Are claims factual and verifiable?
- [ ] **Contact options**: Are support and contact methods easy to find?
- [ ] **Company information**: Is company background transparent?

**Impact**: High - Trust is foundational for conversion
**Priority**: Address trust gaps immediately, especially for new products

---

## 7. Conversion Optimization

### Call-to-Action (CTA) Design
- [ ] **Visual hierarchy**: Are CTAs the most prominent elements?
- [ ] **Clarity**: Is it clear what happens when clicking the CTA?
- [ ] **Action-oriented copy**: Do CTAs use strong action verbs?
- [ ] **Contrast**: Do CTAs stand out visually?
- [ ] **Placement**: Are CTAs positioned at decision points?

### Anchoring & Pricing
- [ ] **Anchoring effect**: Are price comparisons or savings shown?
- [ ] **Framing**: Is pricing framed advantageously ($/month vs $/year)?
- [ ] **Value emphasis**: Are benefits highlighted near pricing?
- [ ] **Tiered options**: Are pricing tiers clearly differentiated?

### Friction Reduction
- [ ] **Form length**: Are forms as short as possible?
- [ ] **Social signup**: Are social login options available?
- [ ] **Guest checkout**: Can users complete transactions without accounts?
- [ ] **Auto-fill support**: Do forms support browser auto-fill?
- [ ] **Error prevention**: Are errors prevented through validation?

**Impact**: Very High - Direct impact on conversion rates
**Priority**: Highest priority for business-critical flows

---

## 8. Mobile & Responsive Experience

### Mobile-Specific Considerations
- [ ] **Touch targets**: Are buttons and links large enough (44x44px minimum)?
- [ ] **Cognitive load**: Is content even more focused on mobile?
- [ ] **Progressive disclosure**: Is mobile UI simplified appropriately?
- [ ] **Thumb zones**: Are critical actions in easy-to-reach areas?
- [ ] **Performance**: Does mobile load quickly on slower connections?

### Cross-Device Consistency
- [ ] **Familiarity**: Are platform conventions followed (iOS/Android)?
- [ ] **Feature parity**: Are key features available on all devices?
- [ ] **Responsive design**: Does layout adapt gracefully to screen sizes?

**Impact**: High - Mobile often represents 50%+ of traffic
**Priority**: Test on actual devices, not just browser emulation

---

## 9. Accessibility & Inclusion

### Universal Design
- [ ] **Color contrast**: Does text meet WCAG AA standards (4.5:1)?
- [ ] **Color dependency**: Is information conveyed beyond color alone?
- [ ] **Keyboard navigation**: Can all functions be accessed via keyboard?
- [ ] **Screen reader support**: Are images and interactive elements properly labeled?
- [ ] **Text sizing**: Can text be resized without breaking layout?

### Cognitive Accessibility
- [ ] **Plain language**: Is copy clear and jargon-free?
- [ ] **Consistent patterns**: Are UI patterns predictable?
- [ ] **Error tolerance**: Are errors easy to recover from?
- [ ] **Time limits**: Are time limits adjustable or removable?

### Inclusive Content
- [ ] **Diverse representation**: Do images/examples represent diverse users?
- [ ] **Inclusive language**: Is language welcoming to all?
- [ ] **Alternative paths**: Are there options beyond gamification/competition?

**Impact**: High - Affects usability for 15-20% of users
**Priority**: Address critical accessibility gaps (color contrast, keyboard nav) immediately

---

## 10. Performance & Technical UX

### Load Time & Perceived Performance
- [ ] **Initial load**: Does the page load in under 3 seconds?
- [ ] **Progressive loading**: Is content rendered progressively?
- [ ] **Skeleton screens**: Are placeholders shown during load?
- [ ] **Image optimization**: Are images compressed and lazy-loaded?
- [ ] **Critical CSS**: Is above-the-fold content prioritized?

### Interaction Performance
- [ ] **Responsiveness**: Do interactions feel instant (<100ms)?
- [ ] **Animation performance**: Are animations smooth (60fps)?
- [ ] **Loading states**: Are loading indicators shown for slow operations?

**Impact**: High - Performance directly affects bounce rate
**Priority**: Optimize critical rendering path first

---

## Prioritization Framework

After completing the checklist, prioritize issues using this matrix:

### Critical (Fix Immediately)
- Broken core functionality
- Major trust/security concerns
- Severe accessibility violations
- Critical performance issues (>5s load)

### High Priority
- Navigation confusion
- High cognitive load in key flows
- Missing critical social proof
- Mobile usability problems
- Poor CTA visibility

### Medium Priority
- Aesthetic inconsistencies
- Missing persuasion elements
- Moderate performance issues
- Engagement mechanic improvements

### Low Priority
- Nice-to-have gamification
- Advanced personalization
- Subtle copy optimizations
- Visual polish details

## Reporting Template

After completing the analysis:

### Executive Summary
- **Overall assessment**: [1-2 sentences]
- **Top 3 strengths**: What's working well
- **Top 3 opportunities**: Biggest impact improvements
- **Estimated impact**: Potential improvement to key metrics

### Detailed Findings

For each category:
1. **Current state**: What's working, what's not
2. **Psychology principles applied/missed**: Reference specific effects
3. **Specific recommendations**: Actionable improvements
4. **Expected impact**: High/Medium/Low
5. **Implementation effort**: Hours/days/weeks estimate

### Prioritized Action Plan

1. **Quick wins** (High impact, low effort)
2. **Strategic improvements** (High impact, high effort)
3. **Optimizations** (Medium impact, medium effort)
4. **Future considerations** (Lower priority items)

---

## A/B Testing Recommendations

Based on findings, suggest tests in priority order:

1. **Highest impact**: CTA copy/placement, pricing anchors
2. **High impact**: Social proof placement, urgency messaging
3. **Medium impact**: Visual hierarchy adjustments, copy framing
4. **Experimental**: Gamification elements, personalization

For each test:
- **Hypothesis**: Based on psychological principle
- **Test setup**: Control vs variant description
- **Success metrics**: Primary and secondary KPIs
- **Sample size needed**: Based on current traffic
- **Duration**: Estimated time to significance

---

This checklist should be adapted based on the specific product, audience, and business goals. Not all items apply to all situations—use judgment to focus on the most relevant factors.
